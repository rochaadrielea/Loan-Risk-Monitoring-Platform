DAX Design Decisions

Power BI Measure Architecture · April 2026 · Adriele Rocha Weisz

Foundation

Three Design Principles Applied to Every Measure

01
LN Filter in DAX

Applied inside CALCULATE (not in Power Query) so it holds regardless of what new product types appear in future datasets.

02
DIVIDE() not /

Zero-safe division. Returns BLANK instead of error when the denominator is zero. Never breaks charts.

03
LASTNONBLANK for FX/Prices

Adapts to any date range automatically. No hardcoded rates. Works when fx and prices tables have different date coverage than transactions.

Measure 1

Loan Exposure

Net loan value from LN-type transactions only. The VAR pattern eliminates floating point artifacts that appear when buys and sells cancel out exactly.

transactions table DAX 
Loan Exposure =
VAR Result =
    ROUND(
        CALCULATE(
            SUMX(
                transactions,
                transactions[transaction-quantity] *
                transactions[transaction-price]
            ),
            products[product-type] = "LN"
        ),
        2
    )
RETURN
    IF(Result = 0, BLANK(), Result)
Why VAR + BLANK()
Floating point arithmetic produces values like 5.8E-11 when buys and sells cancel exactly. ROUND fixes display but not logic: a rounded 0.00 is still not BLANK and Power BI still renders the bar. VAR + IF(Result = 0, BLANK()) removes the bar at source.
Why LN in CALCULATE
Filtering in CALCULATE (not Power Query) means the filter applies dynamically. If a new product type arrives in a future dataset, the measure still returns only LN exposure with zero model changes.
Measure 2

Loan Exposure %

Loan exposure expressed as a percentage of total portfolio value. The IF wrapper makes closed positions return BLANK (consistent with Loan Exposure behaviour).

transactions table DAX 
Loan Exposure % =
VAR Result =
    DIVIDE(
        [Loan Exposure],
        [Portfolio Value]
    )
RETURN
    IF(Result = 0, BLANK(), Result)
Why DIVIDE()
DIVIDE() returns BLANK when Portfolio Value is zero or BLANK. Using / would return an error and break the visual. DIVIDE() is always correct here.
Why IF wrapper
When [Loan Exposure] is BLANK, DIVIDE returns BLANK automatically. The IF(Result = 0, BLANK()) handles the edge case where both measures return a valid zero (no net position). Keeps behaviour consistent with Loan Exposure.
Measure 3

Avg LTV

Average loan-to-value ratio for LN products, excluding statistical outliers. The IQR fence is computed dynamically inside the measure so no separate column is needed for filtering.

transactions table DAX 
Avg LTV =
VAR Q1 = PERCENTILE.INC(products[loan-property-value-ratio], 0.25)
VAR Q3 = PERCENTILE.INC(products[loan-property-value-ratio], 0.75)
VAR IQR = Q3 - Q1
VAR UpperFence = Q3 + 3 * IQR
VAR LowerFence = Q1 - 3 * IQR
RETURN
CALCULATE(
    AVERAGEX(
        FILTER(products,
            products[product-type] = "LN"
            && products[loan-property-value-ratio] <= UpperFence
            && products[loan-property-value-ratio] >= LowerFence
        ),
        DIVIDE(1, products[loan-property-value-ratio])
    )
)
LTV Formula
The source column is loan-property-value-ratio (the inverse of LTV). LTV = 1 / ratio. DIVIDE() handles the zero case. Raw data is never modified.
Outlier Exclusion
PR-P4N4N has a ratio of 33.33 (LTV of 0.03). Including it pulls the average to a meaningless number. The 3xIQR fence excludes it dynamically without hardcoding a threshold.
Consistency with Pipeline
The 3xIQR method matches the Python pipeline exactly. Both tools flag and exclude the same records. The outlier-flag calculated column on the products table uses the same formula for visual flagging.
Measure 4

Avg LTV %

Avg LTV scaled to percentage for display on P4 (Metrics %). Formatting is a visual property, not a calculation change.

transactions table DAX 
Avg LTV % = [Avg LTV] * 100
Why x100 not format
P3 uses the decimal measure (0.83) with a fixed gradient at 0 / 0.8 / 1. P4 needs the percentage value (83) with the gradient at 0 / 80 / 100. Two separate visuals require two separate scales. One measure, one multiplication.
Measure 5

Portfolio Value

Total portfolio value combining P1-EUR and P2-USD at the most recent available price date, with FX conversion applied to the EUR portfolio.

transactions table DAX 
Portfolio Value =
VAR SelectedDate = MAX(transactions[transaction-date])
VAR P1Value =
    CALCULATE(
        LASTNONBLANK(prices[P1-EUR], 1),
        prices[price-date] <= SelectedDate
    )
VAR P2Value =
    CALCULATE(
        LASTNONBLANK(prices[P2-USD], 1),
        prices[price-date] <= SelectedDate
    )
VAR FXRate =
    CALCULATE(
        LASTNONBLANK(fx[EURUSD], 1),
        fx[price-date] <= SelectedDate
    )
RETURN
    (P1Value * FXRate) + P2Value
Why LASTNONBLANK
fx and prices have different date coverage than transactions. A hard relationship breaks when date ranges don't align across dataset swaps. LASTNONBLANK retrieves the most recent available value as of SelectedDate with no model changes required.
FX Conversion
P1 is priced in EUR. P2 is priced in USD. (P1Value x FXRate) converts EUR to USD. Adding P2Value gives total portfolio value in USD. The fx table has 5 null EURUSD rows: Power Query Fill Down handles these before the model loads.
Calculated Column

outlier-flag (products table)

Marks each product as FLAGGED or OK based on the 3xIQR fence. Used in the Overview page flagged products table and as a visual indicator on P3.

products table (calculated column) DAX 
outlier-flag =
VAR Q1 = PERCENTILE.INC(products[loan-property-value-ratio], 0.25)
VAR Q3 = PERCENTILE.INC(products[loan-property-value-ratio], 0.75)
VAR IQR = Q3 - Q1
RETURN
IF(
    products[loan-property-value-ratio] > Q3 + 3 * IQR
        || products[loan-property-value-ratio] < Q1 - 3 * IQR,
    "FLAGGED", "OK"
)
Column vs Measure
A calculated column stores a row-level label (FLAGGED / OK) that can be used in table visuals and slicers. A measure cannot be used in a table row directly. The Avg LTV measure recomputes the fence inline for filtering: same logic, different purpose.
Summary

All Five Measures

Measure Purpose Key Pattern
Loan Exposure Net loan value, LN products only VAR + BLANK() eliminates floating point artifacts
Loan Exposure % Exposure as % of portfolio value DIVIDE() with IF wrapper for zero positions
Avg LTV Average LTV, outliers excluded 3xIQR fence computed inline via VAR
Avg LTV % Avg LTV scaled for % visuals on P4 [Avg LTV] x 100 separate scale for gradient
Portfolio Value Total value, both portfolios, FX applied LASTNONBLANK adapts to any date range
April 13 Guarantee

New Dataset Compatibility

Confirmed by Eric Della-Negra (1 April 2026)

"Same data structure. Only data may change." Every measure was built for this. No filters to adjust, no thresholds to recalibrate, no paths to update except the three Power Query parameters. Load new files, click Refresh, dashboard updates. Zero model changes required.

LN filter in CALCULATE Future product types do not affect the measure
3xIQR fence via VAR Outlier threshold recalculates automatically on new data
LASTNONBLANK for fx/prices Date range mismatches handled at DAX level
VAR + BLANK() Ghost bars eliminated at source on any dataset
